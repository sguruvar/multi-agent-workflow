Agent Design

The Supervisor Pattern

A central LLM classifies user intent via structured output and routes to the appropriate specialist agent.

  • LLM-based intent classification (not rules)
  • Each specialist is independently replaceable
  • Session memory persists across turns
  • Framework-agnostic via Python functions
  • Traces show full routing decision chain
class IntentClassification(BaseModel): intent: Literal[ "order_status", "billing", "tech_support", "general" ] reasoning: str classifier = get_chat_model() .with_structured_output( IntentClassification )

Conversational State

Session Memory

Each customer gets a thread — full conversation history is maintained across multiple interactions and agent handoffs.

  • Thread-based isolation (thread_id)
  • Messages accumulate across turns
  • Specialist agents see conversation context
  • Visible in Omni Session Explorer
  • Traces correlate by session
# Each session gets a unique thread config = { "configurable": { "thread_id": "session-alice-001" } } # Multi-turn conversation supervisor_graph.invoke( {"messages": [HumanMessage(msg)]}, config=config ) # Memory persists between calls

AgentCore Runtime

Deploy Your Agent

Wrap your existing graph with BedrockAgentCoreApp — telemetry flows to CloudWatch automatically.

  • Same code runs locally & in production
  • OTEL spans emitted per invocation
  • Intent, latency, tokens all captured
  • Zero additional collector setup
from bedrock_agentcore.runtime import ( BedrockAgentCoreApp ) app = BedrockAgentCoreApp() @app.entrypoint async def invoke(payload): result = supervisor_graph.invoke( {"messages": [ HumanMessage( payload["prompt"] ) ]}, config={...} ) return {"result": result} app.run()
